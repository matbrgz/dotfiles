# i18n — Design Spec

## Goal

Add internationalization to the GUI app with support for English, Portuguese (pt-BR), and Spanish (es). Language preference is persisted in `localStorage`. A language switcher lives in the Header.

---

## Tech Stack

- **Library:** `react-i18next` + `i18next`
- **Translation loading:** Static JSON imports bundled by Vite (no runtime fetch needed for a desktop app)
- **Translation files:** JSON, one file per namespace per language
- **Persistence:** `localStorage` key `dotfiles-lang` (default: `'en'`)

---

## Architecture

### Initialization (`src/i18n.ts`)

A single config file initializes i18next:
- Imports all 27 JSON files statically and passes them to `i18next.init` as `resources`
- Reads initial language from `localStorage.getItem('dotfiles-lang') ?? 'en'`
- Sets `fallbackLng: 'en'` — missing translations fall back to English silently
- Sets `interpolation.escapeValue: false` (React already escapes)

This file is imported once in `main.tsx` (side-effect import) before `<App>` renders. No wrapper component needed — `i18next` is global.

### Language Switching

`Header.tsx` renders a compact dropdown (`<select>` or button group) showing the current language. On change:
1. Call `i18n.changeLanguage(lang)`
2. Persist: `localStorage.setItem('dotfiles-lang', lang)`

### Fallback Behavior

If a key is missing in a translation, i18next renders the key string itself (e.g., `"install"`). No crashes, no blank strings.

## Namespace Structure

One namespace per feature area. Each namespace maps to one JSON file per language.

| Namespace | Components | Scope |
|-----------|-----------|-------|
| `common` | All | Generic buttons (Cancel, Copy), loading/error states, app name |
| `layout` | App, Sidebar, Header | Tab names, nav labels, stats pills, search, sync button |
| `inventory` | InventoryTab | Package list, install/reinstall buttons, empty states |
| `environment` | EnvironmentTab | Dotfiles stats, sync actions |
| `settings` | SettingsTab | Form labels, toggle descriptions, save button |
| `profile` | ProfileTab | Stat cards, environment labels |
| `disk` | DiskCleanerTab | Scan, clean, categories, confirmations |
| `memory` | MemoryTab | RAM stats, process table, kill buttons |
| `git` | GitReposTab, GitRepoDetail | Repos list, card badges, detail panel, branch/stash/tag actions |

### File Layout

```
apps/gui/src/locales/

    profile.json

  es/       (same structure)
```

27 files total (9 namespaces × 3 languages).

## Translation Key Conventions

- **snake_case** keys
- **Interpolation** uses `{{varName}}` syntax: `"installedCount": "{{count}} / {{total}} installed"`
- **Pluralization** not needed for current strings (counts are displayed as numbers inline)
- **Context** suffix for button variants: `btn_save`, `btn_saving`, `btn_scan`, `btn_scanning`

### Example: `en/inventory.json`

```json
{
  "install": "Install",
  "reinstall": "Reinstall",
  "installing": "Installing...",
  "scanning": "Scanning installed packages...",
  "statsInstalled": "{{count}} / {{total}} installed",
  "emptySearch": "No packages match \"{{query}}\""
}
```

### Example: `pt-BR/inventory.json`

```json
{
  "install": "Instalar",
  "reinstall": "Reinstalar",
  "installing": "Instalando...",
  "scanning": "Verificando pacotes instalados...",
  "statsInstalled": "{{count}} / {{total}} instalados",
  "emptySearch": "Nenhum pacote corresponde a \"{{query}}\""
}
```

## Component Integration Pattern

### Before

```tsx
<button>Install</button>
<p>Scanning installed packages...</p>
<p>{count} / {total} installed</p>
```

### After

```tsx
const { t } = useTranslation('inventory');

<button>{t('install')}</button>
<p>{t('scanning')}</p>
<p>{t('statsInstalled', { count, total })}</p>
```

Components that need strings from multiple namespaces call `useTranslation` once per namespace or use `useTranslation(['ns1', 'ns2'])` with the `t('ns1:key')` prefix form.

## Language Switcher (Header)

A compact 3-button toggle in `Header.tsx`, right side, before the Sync button:

```
[ EN ][ PT ][ ES ]
```

Active language is highlighted with primary color. On click: `i18n.changeLanguage(code)` + `localStorage.setItem('dotfiles-lang', code)`.

Languages:
- `en` → label `EN`
- `pt-BR` → label `PT`
- `es` → label `ES`

---

## String Scope

Log messages in the terminal panel (`App.tsx` `addLog` calls) are **out of scope** — they are developer-facing diagnostics, not user UI. Only strings rendered in JSX are translated.

~130 user-visible strings across 9 namespaces.

## Out of Scope

- RTL language support
- Date/number locale formatting (no locale-sensitive formatting in the current UI)
- Translation of log messages in the terminal panel
- Backend/CLI error message translation
- Pluralization rules (all counts displayed inline as numbers)
